# 1.7 Outils de développement intelligents pour Adobe Commerce

## Objectifs d’apprentissage

Dans ce module, vous utiliserez des outils de développement intelligents tels que Cursor.ai pour développer une extension de votre environnement Adobe Commerce as a Cloud Service. L’objectif de cette extension est de transférer les événements de commande entrants vers un point d’entrée tiers. Le transfert d’événement dans Adobe Commerce as a Cloud Service repose sur Adobe I/O App Builder, Adobe I/O Events et Adobe I/O Runtime. La configuration de tous ces services sera assistée par Cursor.ai.

## Conditions préalables

- Accès à Adobe Commerce as a Cloud Service.
- Accès à Adobe Experience Manager as a Cloud Service.
- Un accès à Adobe I/O avec un accès développeur ou administrateur système.
- Accès à Cursor.ai.
- Vous auriez dû terminer ce module, car il existe des dépendances : [Adobe Experience Manager Cloud Service et Edge Delivery Services](./../../../modules/asset-mgmt/module2.1/aemcs.md).
- Vous auriez dû terminer ce module, car il existe des dépendances : [Adobe Commerce as a Cloud Service](./../../../modules/asset-mgmt/module1.5/accs.md).
