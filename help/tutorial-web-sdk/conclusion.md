# Conclusion et prochaines étapes

Félicitations ! Vous avez terminé le tutoriel « Implémentation de Adobe Experience Cloud avec Web SDK ».

Passons rapidement en revue tout ce que vous avez accompli. Vous avez :

* Création d’un schéma à l’aide de groupes de champs standard et personnalisés.
* A configuré un flux de données.
* Création d’une propriété de balise.
* Champs capturés d’une couche de données, transformés en XDM et envoyés à Platform Edge Network.
* Consentement capturé et identités authentifiées.
* Connexion de l’implémentation de Web SDK aux applications Experience Cloud suivantes :
   * Adobe Experience Platform
   * Adobe Analytics
   * Adobe Audience Manager
   * Adobe Target
   * Adobe Journey Optimizer
* Données transférées de Platform Edge Network vers un hook web avec transfert d’événement.
* Validation de la mise en œuvre à l’aide d’Adobe Experience Platform Debugger.

Vous êtes prêt à passer à la phase suivante de votre parcours, à savoir la mise en œuvre de Adobe Experience Cloud sur votre propre site web.

Et il y a toujours plus à apprendre ! Voici quelques suggestions d’autres contenus à développer sur votre implémentation :


* **Déclencher un parcours dans Journey Optimizer**. Les événements que vous avez implémentés sur le site web Luma peuvent être utilisés pour déclencher des parcours. En savoir plus dans ce [tutoriel vidéo](https://experienceleague.adobe.com/en/docs/journey-optimizer-learn/tutorials/journeys/use-case-transactional-journey).
* **Connecter Customer Journey Analytics**. Si vous avez créé le [jeu de données Platform](setup-experience-platform.md), vous pouvez connecter le jeu de données à Customer Journey Analytics. En savoir plus dans ce [tutoriel vidéo](https://experienceleague.adobe.com/en/docs/customer-journey-analytics-learn/tutorials/connections/connecting-customer-journey-analytics-to-data-sources-in-platform)
* **Créer un segment dans Platform**. Si vous avez activé votre [schéma et jeu de données pour le profil client en temps réel](setup-experience-platform.md), vous pouvez créer des segments en fonction de vos événements web, puis envoyer ces segments vers des destinations dans Real-Time Customer Data Platform. En savoir plus sur le créateur de segments dans ce [tutoriel vidéo](https://experienceleague.adobe.com/en/docs/platform-learn/tutorials/audiences/create-audiences).
* **Implémentation de Platform Mobile SDK**. Maintenant que vous maîtrisez un SDK, apprenez-en un autre ! Adobe Experience Platform Mobile SDK est utilisé pour alimenter Experience Cloud et les services tiers sur les applications mobiles. Il existe un [tutoriel pratique similaire pour Mobile SDK](https://experienceleague.adobe.com/en/docs/platform-learn/implement-mobile-sdk/overview). Effectuez les deux et voyez les profils fusionner sur tous les appareils !
* **En savoir plus sur Experience Platform**. Découvrez comment ingérer des données provenant d’autres sources dans [Prise en main de Adobe Experience Platform pour les architectes et ingénieurs de données](https://experienceleague.adobe.com/en/docs/platform-learn/getting-started-for-data-architects-and-data-engineers/overview)
